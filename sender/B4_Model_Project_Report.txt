ABSTRACT
====================================================================
Intracranial hemorrhage (ICH) is a time-critical emergency where delayed detection
can significantly increase mortality and long-term neurological disability. This
project develops an end-to-end AI-assisted screening pipeline for non-contrast
head CT, with strong focus on three practical requirements: calibrated confidence,
explainability, and structured triage communication.

The implemented system accepts DICOM studies, applies CT-window preprocessing,
builds a 2.5D context representation (previous-current-next slices), runs
EfficientNet-B4 inference, calibrates probabilities, and produces standardized
machine-generated reports with confidence band and triage recommendation. A
labeled evaluation protocol was run on 500 RSNA-format samples using image-level
any labels.

For the final single-model configuration (B4 Fold 4), measured results were:
1. ROC-AUC: 0.95974
2. PR-AUC: 0.84902
3. Brier Score: 0.05450
4. Expected Calibration Error (ECE, 10-bin): 0.01764
5. Sensitivity: 0.87952
6. Specificity: 0.90647
7. Positive Predictive Value (PPV): 0.65179
8. Negative Predictive Value (NPV): 0.97423

These results indicate high discrimination and strong calibration, supporting use
as a triage-assistive model. The project contribution is not only model accuracy,
but also clinically interpretable reporting behavior aligned with safe deployment
principles.

Keywords: Intracranial Hemorrhage, Head CT, EfficientNet-B4, 2.5D, Calibration,
Grad-CAM, Clinical AI, Triage

1. INTRODUCTION
====================================================================
Intracranial hemorrhage is one of the most urgent radiological findings in
emergency medicine. Rapid and accurate screening is critical because hematoma
expansion and rising intracranial pressure can worsen patient outcomes within
hours. Non-contrast head CT remains the frontline imaging modality due to speed,
availability, and sensitivity for acute bleeding.

Despite its importance, practical workflow challenges remain:
1. High emergency throughput causes reporting delays.
2. Screening burden often falls on junior readers during peak hours.
3. Subtle hemorrhage patterns can be missed in time-constrained environments.
4. Pure prediction models without confidence context are unsafe in clinical use.

Therefore, this project focuses on building a complete screening-assistive
pipeline, not only a classification model.

5. SYSTEM DESIGN AND METHODOLOGY
====================================================================
5.1 End-to-End Workflow
1. Load DICOM study/slices.
2. Sort slices by instance metadata.
3. Build adjacent context window (prev, center, next).
4. Apply CT windowing to each slice.
5. Stack 3 windows x 3 adjacent slices = 9 channels.
6. Normalize using training statistics.
7. Run B4 model inference.
8. Apply calibration and thresholding.
9. Generate explainability heatmap (if enabled).
10. Produce structured JSON and tabular outputs.

5.2 CT Windowing Used
1. Brain window: WL 40, WW 80
2. Subdural window: WL 75, WW 215
3. Soft tissue window: WL 40, WW 380

5.3 2.5D Representation Rationale
Single-slice prediction can miss continuity patterns. 2.5D local context preserves
neighboring anatomical continuity without requiring heavy full-3D architectures.

6.2 Input Adaptation
Original EfficientNet input stem is adapted for 9-channel input to consume 2.5D

6.3 Output Head
Model outputs 6 logits:
1. any
2. epidural
3. intraparenchymal
4. intraventricular
5. subarachnoid
6. subdural

implemented and tested separately but is intentionally excluded from final claim.

11. DISCUSSION
====================================================================
11.1 What Improved from Earlier Project Stage
1. Better feature context via 2.5D (9-channel) input
2. Higher-capacity B4 representation vs lightweight baseline
3. Reliable calibration (very low ECE)
4. Standardized triage-aware output language
5. Explainability integration in the B4 reporting path

11.2 Clinical Relevance
1. High NPV (0.97423) supports safer negative triage in screening workflow.
2. Specificity above 0.90 reduces unnecessary high-priority alerts.
3. Sensitivity remains high but non-perfect, reinforcing mandatory human oversight.

11.3 Operational Trade-off
B4 fold4 is slower than B0 on CPU, but quality gains are substantial. For practical
use, GPU inference and selective heatmap generation are recommended.

11.4 Relation to Inspiration Literature
The project reflects key literature principles (window-informed input,
explainability, clinical framing), while adding calibration and deployment-facing
report governance needed for real-world student demonstration.
